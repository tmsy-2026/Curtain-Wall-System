# 幕墙工程管理系统

## 系统简介

这是一款面向幕墙施工项目的协同管理平台，支持多角色登录、分模块管理、数据隔离与流程审批。

## 功能特性

### 核心功能模块
- **账号与权限系统**：基于角色的访问控制（RBAC），支持6种角色
- **材料仓管管理**：材料档案、入库、出库、库存查询、盘点
- **劳务管理**：人员档案、班组管理、考勤记录、工资核算
- **财务管理**：收支记录、发票管理、成本核算、对账结算
- **采购管理**：采购申请、订单跟踪、供应商管理、办公用品
- **流程审批**：多类型审批流程（采购、付款、请假、合同）

## 角色说明

| 角色 | 权限范围 |
|------|----------|
| 项目经理 | 全模块管理、账号管理、角色权限配置 |
| 材料仓管员 | 材料档案、入库、出库、库存、审批 |
| 劳务管理员 | 人员档案、班组、考勤、工资、审批 |
| 财务人员 | 收支、发票、成本、对账、审批 |
| 采购人员 | 采购申请、订单、供应商、办公用品、审批 |
| 审批人员 | 审批流程处理 |

## 技术架构

- **前端**：原生 HTML5 + CSS3 + JavaScript
- **UI**：自定义响应式组件库
- **存储**：LocalStorage（单机版）/ 腾讯云开发（云端版）
- **部署**：静态网站托管

## 部署指南

### 方案一：直接部署到静态服务器

将所有文件上传到任意静态服务器即可使用。

### 方案二：腾讯云静态网站托管（推荐）

1. 登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)
2. 创建环境，进入「静态网站托管」
3. 上传所有文件到托管空间
4. 配置自定义域名（可选）

### 方案三：GitHub Pages + Cloudflare（免费方案）

1. 创建 GitHub 仓库，上传代码
2. 启用 GitHub Pages
3. 配置 Cloudflare CDN 加速（解决国内访问问题）

## 数据备份

系统支持手动备份数据：
1. 进入「系统管理」→「数据备份」
2. 点击「导出数据」下载JSON文件
3. 需要恢复时，点击「导入数据」选择备份文件
